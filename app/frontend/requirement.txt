Technology : Electron JS and Java for backend processing
Build using : npm and Gradle
Resultion: Fit in 50% of Screen from the Left of the Windows Width 50% and Height 100%
Requirment 
Phase 1: Build UI to Which having the ATM like View and Device get bind base on the Device Configuraed in #DeviceMap.json 
Where  "DeviceType" : "XFS" those need to display on Screen.
Make emplty space for the ATM window load top left corner 1024x768
Phase 2: Java Processing to Start all the device using XFS Service 
base on the events Animation to display the user like Ready for Insert Card.
Eject Card, Dispancer ready with Cash, Receipt print etc..
Phase 3: Add device wise option to generate faults. that will call the java processing and that interact with XFS Service to perform action.


Commands:

nodemon --ext js,html,css --exec electron .

Criteria for fallback logic:
1. If the fixed screen in our UI is smaller than the device screen the application is running on then check
   1.1 If there is still space remaining for the bottom panel to fit below the inner fixed screen.
       If yes then go ahead with the basic UI design in which elements are arranged in LShape around the inner fixed screen
       If No then move the bottom row elements to right column i.e basically keep the components size same just add another column so there are 2 right columns for this follow the fallback UI image
       So basically there shoudld be no scroll to the window since in both cases we are arranging the compnents in a way where we are able to see all of them
2. If the fixed screen in our UI is larger than the device screen the application is running on then 
   2.1 have the screen of the same fixed size but now we dont follow the LShape design 
   2.2 By default the design is screen with 2 right columns 
   2.3 the complete application window will be of the size of the screen and the components will fit themselves in the scrren height itself
   2.4 Scroll is provided till the height of the screen which is basically the size of the window
